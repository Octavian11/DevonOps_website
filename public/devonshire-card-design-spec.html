Implementation Spec

Card & Component Design

Establish a unified card design system across 7 component types. Fix pricing card hierarchy, standardize internal spacing, and create consistent visual language across persona cards, risk cards, path cards, and proof blocks.

6 implementation itemsApril 2026

Before touching individual card types, establish a shared design vocabulary. Every card on the site should inherit from this token system.

Implementation Summary
Priority sequence

Execute in this order

  • Pass 1 — Tokens (Foundation): Add CSS custom properties to :root. No visual changes yet.
  • Pass 2 — Pricing cards (Item 02): Highest conversion impact. Scale lift, warm shadow, badge, cream bg on recommended card.
  • Pass 3 — Risk cards (Item 03): Replace dashed red borders with top-accent + muted-rose. Most visually noticeable change.
  • Pass 4 — Persona + Path cards (Items 01, 04): Equal-height grids, flex column layout, proof quote tinting. Replace dashed path borders with solid.
  • Pass 5 — Proof blocks (Items 05, 06): Standardize blockquote treatment (#FFFBF0 bg, gold left-border, rounded right corners) across all proof elements.

This creates "same family, different personality" rather than "seven independent designs."
