Expand first 2 representative outcomes by default
The "Representative Outcomes" section currently hides all 4 outcomes behind a toggle. Show the first 2 expanded on page load; keep the remaining 2 behind the toggle.
High Outcome visibility on load

The current toggle reads: "Show 4 representative outcomes (gap → fix → result) ▸". After this change, 2 outcomes will be visible immediately, and the toggle text changes to reflect only the hidden remainder.

Implementation approach

/* Structural change: split outcomes into two groups */ /* Group 1: Always visible (first 2 outcomes) */ <div class="outcomes-visible"> <!-- Outcome 1: Platform Stabilization --> <!-- Outcome 2: Vendor Optimization --> </div> /* Group 2: Hidden behind toggle (remaining 2) */ <div class="outcomes-hidden" id="more-outcomes"> <!-- Outcome 3 --> <!-- Outcome 4 --> </div> <button class="outcomes-toggle" onclick="toggleOutcomes()"> Show 2 more outcomes ▸ </button>
.outcomes-visible { display: flex; flex-direction: column; gap: 16px; } .outcomes-hidden { max-height: 0; overflow: hidden; transition: max-height 0.3s ease; } .outcomes-hidden.expanded { max-height: 2000px; /* generous max to accommodate content */ } .outcomes-toggle { background: none; border: 1px solid #D6D4CE; border-radius: 8px; padding: 12px 24px; font-size: 14px; font-weight: 600; color: #1B2A4A; cursor: pointer; margin-top: 16px; transition: border-color 0.2s; } .outcomes-toggle:hover { border-color: #C8952E; } /* Mobile: show only 1 outcome by default */ @media (max-width: 768px) { .outcomes-visible .outcome-card:nth-child(n+2) { display: none; } /* Move 2nd outcome into hidden group via JS, or use CSS-only approach above */ }
JS for the toggle: function toggleOutcomes() { document.getElementById('more-outcomes').classList.toggle('expanded'); } — or adapt to whatever toggle/accordion pattern your site framework uses.
