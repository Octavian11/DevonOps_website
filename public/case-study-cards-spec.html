Incident Instability
Gap

Same failures recurring every 4–6 weeks. No severity model, no ownership, no postmortems.

Fix

Severity model, incident command, escalation paths, postmortem cadence.

Result

~67% incident reduction. ~31% faster resolution. Board reporting shifted from crisis-driven to weekly structured reviews.

Change-Driven Outages
Gap

Deployments causing outages. No change calendar, no risk classification, no rollback plans.

Fix

CAB-lite process, risk classification, rollback discipline, change-incident correlation.

Result

~60% fewer critical outages. Uptime: 94% → 99%. Change-incident correlation visible to the board within 30 days.

Board Reporting / KPI Ambiguity

Fix

KPI library, weekly operating review, executive dashboard, board-ready reporting pack.

Result

Weekly operating rhythm installed. Issues surfaced through cadence, not crisis. Board gained real-time ops visibility.

Implementation Notes for Coding Agent

Structure change (homepage): Replace the current .track-vignettes-grid (two side-by-side flex cards with dense paragraphs) with stacked full-width .case-card components. Each card uses a two-column grid for Situation + Intervention, then a four-column metrics strip, then a PE Translation footer. The additional outcome cards (currently in a separate .outcomes-visible section) are nested inside each parent case card as expandable content.
Structure change (about page): Keep the expanded Context / What I Built / Measured Outcomes / PE Translation format, but replace the Measured Outcomes prose paragraph with the same .case-metrics four-tile strip used on the homepage. This creates visual consistency across pages while allowing the About page to retain its longer narrative.
Metrics are not prose. The single most important change is breaking outcome numbers out of sentences and into visual tiles. Current: "Zero critical outages for 18 consecutive months. Availability: 94% → 99.2%. MTTR reduced 31%." New: four tiles with large numbers and short labels. PE buyers scan numbers before they read sentences.
Metric numbers: single color only. Each metric number renders entirely in midnight blue (#1B2A4A). Do NOT split numbers and their units into two colors (e.g., "28" in blue + "%" in gold). This was visually distracting and fragmented. Gold accent is applied structurally via the 2px gold top-border on the .case-metrics strip, not on individual characters.
Mobile behavior: On screens below 680px, the Situation/Intervention grid collapses to single column. The metrics strip collapses to a 2×2 grid. The PE Translation footer stacks vertically. The outcome cards grid goes single column. The proof strip at the top goes single column with horizontal dividers.
Expandable outcomes: The "Related outcomes" toggle uses a simple JS handler (below). Default state is collapsed. The chevron rotates 90° on open. Animation is CSS transition on transform.
