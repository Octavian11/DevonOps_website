Item 01
Remove sticky CTA bar on ALL viewports
The mobile spec already hid the sticky bar for ≤768px. This extends the removal to desktop. The bar overlaps the footer, competes with inline CTAs, and creates banner blindness.
Critical Kill sticky bar globally

The current mobile media query uses display: none !important for ≤768px. Extend this to all viewport widths by moving the rule outside the media query, or by removing the element from the DOM entirely.

Option A — CSS-only (recommended if you want easy reversal)

/* Remove the @media wrapper — apply globally */ .sticky-cta-bar, [class*="sticky-bottom"], [class*="fixed-cta"] { display: none !important; }

Option B — DOM removal (cleaner, no dead CSS)

Delete the entire sticky bar HTML element from the page template. This is the cleaner approach if you're confident you won't want it back.

Why not redesign instead of remove? The page already has 3 inline "Book a Fit Check" buttons (nav, hero, post-pricing) plus the footer form. Adding a 4th persistent one creates saturation. PE buyers browsing a consulting site are not impulse purchasers — they don't need a "buy now" bar following them like an e-commerce checkout prompt. The sticky bar pattern signals mass-market SaaS, not premium advisory.
Item 02
Downgrade persona card CTAs from buttons to text links
The three buyer-type cards (Independent Sponsors, LMM PE Funds, Family Offices) each have a full-sized CTA button at the bottom. These cards trigger a moment of self-identification — "this is for me" — which is a good moment for a CTA. But three competing full-sized buttons create visual noise. Downgrade all three to subtle text links that preserve the conversion path without competing with the primary CTAs in the hero and pricing sections.

Key visual distinction: The secondary button now has a gray border (#D6D4CE) instead of gold, and midnight text instead of gold text. This creates an unmistakable visual hierarchy — gold filled button is clearly the primary action, gray-outlined button is clearly the alternative. Currently both use gold styling which flattens the hierarchy.

Implementation Summary
Priority sequence

Execute in this order

  • Pass 1 — Removals (Items 01, 03): Kill the sticky bar globally, remove CTA pair from levers section. These are pure deletions — fast, low-risk, no conversion paths lost.
  • Pass 2 — Downgrades (Items 02, 04): Convert all persona card CTA buttons and both path card CTA buttons into text links using the .card-text-link class. Every conversion path is preserved — the visual weight shifts from buttons to links.
  • Pass 3 — Styling (Item 05): Apply the primary/secondary CTA visual hierarchy across all remaining buttons. Key change: secondary buttons get gray borders (#D6D4CE) instead of gold, and midnight text instead of gold text.
  • Pass 4 — Copy (Item 06): Update the footer availability copy to add "Book early."
