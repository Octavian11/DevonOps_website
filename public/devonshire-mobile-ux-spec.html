Issue 01
Sticky CTA bar steals 15% of the viewport
The persistent bottom bar with "Book a Fit Check" + "Score Your Deal →" appears on scroll and never leaves. On a 390×844 screen the bar + safe area consume ~120px — 14.2% of usable viewport.
Critical Sticky bar dimensions & behavior

The bar currently renders as a fixed-position element at the bottom of the viewport. It contains a line of persuasive text ("Ready to find the gaps?"), two side-by-side CTA buttons, and a small dismiss "×" icon. On mobile, all this content wraps and stacks partially, creating visual clutter.

PropertyCurrentFixNotes
Total bar height ~120px 0px (remove) Kill the sticky bar entirely on mobile
Viewport consumed 14.2% 0% Reclaimed for content
Trigger point ~500px scroll n/a Was appearing after hero section
Z-index 999+ n/a Was overlapping footer content

Specification

/* Hide sticky bar on viewports ≤ 768px */ @media (max-width: 768px) { .sticky-cta-bar, [class*="sticky-bottom"], [class*="fixed-cta"] { display: none !important; } }
Why remove instead of redesign? The page already has 6–7 inline CTA placements (hero, persona cards, levers section, pricing, pre-footer). The sticky bar is redundant — every scroll position is within 2 viewport-heights of a CTA. Removing it reclaims 120px on every screen, eliminates the footer overlap bug, and reduces "banner blindness" pressure.

Implementation Summary
Priority sequence for your coding agent

Execute in this order

  • Pass 1 — Structural (Issues 01, 06): Remove sticky bar, kill section-wrapper borders, switch to full-bleed + dividers. These are global CSS changes that affect every section downstream.
  • Pass 2 — Card system (Issues 02, 03, 04): Apply consistent padding/gap/border-radius system across persona, risk, and pricing cards. The three card types should share a spacing vocabulary: 20px gap, 24px top padding, 28px bottom padding, 12px border-radius.
  • Pass 3 — Interactive elements (Issues 05, 09): Horizontal scroll on pills, accordion row sizing. These are isolated component fixes.
  • Pass 4 — Polish (Issues 07, 08, 10): Header height, metric font sizing, footer form inputs. Low-risk cosmetic adjustments.
Breakpoint strategy: All specs use max-width: 768px as the single mobile breakpoint. If your site uses a different breakpoint (e.g., 640px or 480px), substitute accordingly. The measurements assume a 390px reference viewport (iPhone 14/15) but work across 360–428px range without additional adjustments.
